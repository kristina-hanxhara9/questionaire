# Bundled sample assets

These three files demonstrate the conventions the server expects. They are generated by `scripts/generate_samples.py`. Run that script once after install (on Windows: `py scripts\generate_samples.py`) to materialize them.

## `sample_channel_guide.xlsx`

The Excel guide that drives content. Conventions:

- **One sheet per channel.** Sheet name is the channel name as exposed by `list_channels()`.
- **Header row 1**, columns (case-insensitive):
  - `section` — grouping label; consecutive identical values stay in the same section.
  - `question_type` — controlled vocabulary: `single_choice | multi_choice | scale | open_text | numeric | yes_no | rating`.
  - `guidance` — instruction to the LLM about what to ask and why.
  - `example` — optional exemplar wording.
  - `required` — truthy: `yes | y | true | 1` → required.
  - `notes` — optional author notes (ignored by the renderer).
- Sheets prefixed with `_` (e.g. `_meta`) are skipped by the parser — use them for free-form notes.

## `sample_template.pdf`

The PDF is a **layout reference only**. The parser extracts:

- Section headings (font size > median × 1.2, or bold runs).
- `{{KEY}}` placeholders. Reserved keys: `COMPANY_NAME`, `QUESTIONNAIRE_TITLE`, `LANGUAGE`, `COUNTRY`, `GENERATED_DATE`.
- Optional `<<SECTIONS_START>>` / `<<SECTIONS_END>>` markers (informational).

The PDF is *not* used for rendering. `sample_template.docx` is.

## `sample_template.docx`

The actual renderable template — a [docxtpl](https://docxtpl.readthedocs.io/) Jinja2 .docx that mirrors the PDF visually. The renderer fills it with the JSON the Copilot agent assembled.

Variables exposed in the template:

| Variable | Source |
|---|---|
| `{{ company_name }}` | `RenderRequest.company_name` |
| `{{ questionnaire_title }}` | `RenderRequest.title` |
| `{{ language }}` | ISO language code |
| `{{ country_code }}` | ISO country code or `None` |
| `{{ generated_date }}` | Locale-formatted via Babel |
| `{{ sections }}` | List of `{ heading, questions: [{ text, type, options, required }] }` |

Add custom placeholders by passing `extra_placeholders={"KEY": "value"}` and referencing `{{ KEY }}` in the template.

## Replacing with your own assets

Set the corresponding env var to an absolute path:

```
EXCEL_GUIDE_PATH=C:\Users\you\market-research\my_channels.xlsx
PDF_TEMPLATE_PATH=C:\Users\you\market-research\my_template.pdf
DOCX_TEMPLATE_PATH=C:\Users\you\market-research\my_template.docx
```

Restart the server. Missing files fall back to the bundled samples with a structured warning log.
